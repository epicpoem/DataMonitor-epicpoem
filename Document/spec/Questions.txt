================================================================================
반도체 시료 생산주문관리 시스템 - 스펙 확인 질문 목록
================================================================================

[HIGH - 구현 핵심 로직]

Q1. 재고 차감/추가 시점
    - 승인 시 재고 충분 → CONFIRMED 전환될 때, 해당 수량이 즉시 물리적 재고에서 차감되나요?
    - RELEASE 처리 시 출고 수량이 물리적 재고에서 차감되나요?
    - PRODUCING 완료 → CONFIRMED 전환 시, 생산된 수량이 물리적 재고에 추가되나요?
    → 정책 결정 : CONFIRMED 처리시 차감하지 않고 RELEASED 처리시 차감 (CONFIRMED-RELEASED 처리는 동시간 처리)
                       RELEASE 처리 완료 후 차감
                       PRODUCING 진행 중 생산량 재고 실시간 지속 반영

Q2. 부족분 계산 시 "현재 재고" 기준
    → 정책 결정: 실제(물리적) 재고만 사용. PRODUCING 중인 수량은 물리적 재고에 실시간 산입 (정률 생산으로 가정)
       부족분 = 주문 수량 - 실제 재고

Q3. 가용 재고 공식의 "필요 수량" 활용처
    → 정책 결정: 가용 재고 개념 미사용. 실제 재고만 기준으로 판단. (확인 불필요)

Q4. 생산 완료 감지 방식
    - 생산 완료를 어떻게 감지하나요?
    - 사용자가 메뉴에 진입할 때마다 경과 시간을 체크하여 완료 처리하는 방식인가요?
    - 아니면 별도 스레드/타이머 방식인가요?
    → 메뉴에 진입할 때마다 경과 시간을 체크하여 완료 처리/실시간 생산량 재고 반영 하여도 무방

--------------------------------------------------------------------------------

[MEDIUM - 기능 범위]

Q5. 모니터링 재고 상태(여유/부족/고갈) 판단 기준
    - "주문 대비 재고"에서 어떤 상태의 주문과 비교하나요?
      (RESERVED만? CONFIRMED만? PRODUCING 포함? 전체?)
    → 실제 재고 기준 와 비교
    → RESERVED는 REJECTED 될 수 있으므로 주문 대비 재고에 고려하지 않음
    → CONFIRMED, PRODUCING 상태의 주문 총량과 비교하여 판단

Q6. 시료 ID 생성 방식
    - 시료 등록 시 ID(예: S-001)는 사용자가 직접 입력하나요,
      아니면 시스템이 자동 부여하나요?
    → 시료 등록시 속성 값이 시료 ID, 이름, 평균 생산시간, 수율 임. 이 속성값은 사용자가 입력함. 
    → 기존 ID와 중복시 재입력을 유도하는 에러처리 필요
    → 신규 시료 등록 시 초기 재고 값은 0으로 가정

Q7. 주문번호 채번 규칙
    - 예시 형식: ORD-20260416-0043
    - 뒤의 번호(0043)가 날짜별 순번인가요, 전체 누적 순번인가요?
    → 날짜별 순번으로 하여도 무방 (일별 주문이 10000건은 되지 않음 가정)

Q8. Dummy 데이터 생성 Tool 저장 대상
    - 스펙에 "연결된 DB에 추가"라고 명시되어 있으나,
      데이터 영속성 방식은 개인 자유 선택(파일/JSON/DB)입니다.
    - "연결된 저장소에 추가"로 해석해도 되나요?
    → 저장소에 추가로 해석해도 됨. 본 프로젝트에서는 JSON으로 정책 결정

--------------------------------------------------------------------------------

[LOW - 구현 자유도 확인]

Q9. 시료 검색 가능 속성 범위
    - "이름 등 속성으로 검색"이라고 명시되어 있는데,
      이름 외에 ID, 수율 등 다른 속성도 검색 대상인가요?
    - 이름 검색만으로 충분한가요?
    → 다른 속성도 검색 대상, 검색 대상 선택 가능하도록 개발

Q10. 동일 시료 여러 PRODUCING 주문 처리
    - 동일 시료에 대해 여러 PRODUCING 주문이 대기 중일 때,
      가용 재고 계산 시 해당 시료의 예정 입고 수량을 모두 합산하나요?
    → 예정 수량은 고려하지 않음. 실제 재고(물리적 재고)만 고려
