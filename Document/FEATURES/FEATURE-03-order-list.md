# FEATURE-03: 주문 전체 조회 (선택 구현)

---

## 기능 목적

DataPersistence에 저장된 모든 주문 데이터를 콘솔에서 표 형식으로 출력한다.

---

## 사용자 시나리오

1. 사용자가 메인 메뉴에서 `[2] 주문 전체 조회`를 선택한다.
2. 시스템이 저장소에서 최신 주문 데이터를 읽는다.
3. 시스템이 전체 주문 목록을 표 형식으로 출력한다.
4. 표 하단에 총 건수를 표시한다.
5. 사용자가 확인 후 메인 메뉴로 돌아간다.

---

## 출력 항목

| 항목 | 설명 |
|------|------|
| 주문번호 | 주문 고유 식별자 (예: ORD-20260416-0043) |
| 시료명 | 주문한 시료 이름 |
| 고객명 | 주문 고객 이름 |
| 수량 | 주문 수량 (단위: ea) |
| 상태 | 주문 현재 상태 (RESERVED / PRODUCING / CONFIRMED / RELEASE) |

---

## 예외 처리 기준

- 저장된 주문이 없는 경우: "등록된 주문이 없습니다." 메시지를 출력한다.
- 데이터 파일을 읽을 수 없는 경우: 오류 원인을 안내하는 메시지를 출력하고 메인 메뉴로 복귀한다.
