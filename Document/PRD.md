# PRD (Product Requirements Document)
# DataMonitor - 데이터 모니터링 Tool

작성일: 2026-06-12

---

## 개요

관리자가 DataPersistence에 저장된 반도체 시료 데이터를 콘솔에서 즉시 조회할 수 있는 독립적인 모니터링 도구이다.

---

## 사용자

- **관리자**: 시스템에 저장된 시료 데이터를 확인해야 하는 운영 담당자

---

## 사용자 요구사항

### 핵심 요구사항

1. 사용자는 콘솔에서 프로그램을 실행하면 메인 메뉴를 볼 수 있어야 한다.
2. 사용자는 메인 메뉴에서 원하는 조회 기능을 선택할 수 있어야 한다.
3. 사용자는 프로그램을 종료할 수 있어야 한다.

### 시료 조회

4. 사용자는 등록된 모든 시료의 목록을 한 화면에서 조회할 수 있어야 한다.
5. 사용자는 각 시료의 ID, 이름, 평균 생산시간, 수율, 현재 재고를 확인할 수 있어야 한다.
6. 사용자는 조회 결과에서 전체 시료 건수를 확인할 수 있어야 한다.
7. 사용자는 저장된 시료가 없을 경우 데이터가 없음을 안내받아야 한다.

### 주문 조회 (선택 구현)

8. 사용자는 등록된 모든 주문의 목록을 조회할 수 있어야 한다.
9. 사용자는 각 주문의 주문번호, 시료명, 고객명, 수량, 상태를 확인할 수 있어야 한다.
10. 사용자는 조회 결과에서 전체 주문 건수를 확인할 수 있어야 한다.

### 데이터 최신성

11. 사용자는 프로그램 실행 시점에 저장소에 기록된 최신 데이터를 조회할 수 있어야 한다.
12. 사용자는 데이터 파일이 존재하지 않거나 읽기 실패 시 오류 메시지를 통해 원인을 안내받아야 한다.

---

## 비기능 요구사항

- 조회 결과는 표(table) 형식으로 가독성 있게 출력되어야 한다.
- DataPersistence 저장소의 코드를 직접 수정하지 않고 데이터 파일만 참조한다.
- DataPersistence와 분리된 독립 Repository로 유지한다.
